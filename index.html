Comparing countries fairly using per-capita daily case increments

A walk-through of the full data pipeline — from raw Johns Hopkins case counts to a ranked, population-normalized comparison of countries.
Author

Josef Richter

Published

March 2023

The question

During the pandemic, raw case counts dominated the headlines — but a raw count of “new cases today” tells you almost nothing on its own. A country with 10 million people and one with 80 million will report wildly different absolute numbers even when the outbreak is, proportionally, identical.

This analysis answers a fairer question:

On a given day, which countries had the most new COVID-19 cases relative to their population — and where did Czechia sit in that ranking?

To get there we need three things: a reliable daily case series, population data to normalize by, and a smoothing method to cut through reporting noise (weekends, batch-uploaded backlogs, etc.).

Note

The interactive version of this analysis — with sliders for the averaging window, date, and country selection — is available as a live dashboard. This page shows the full method and reasoning behind it.

The data

Two sources:

  • Confirmed cases — the Johns Hopkins CSSE global time series, the de-facto standard during the pandemic. One row per region, one column per day, cumulative counts. Pulled live over HTTPS.
  • Population — UN population estimates (PopTotal, in thousands), used to convert absolute case counts into per-100k rates.

The core metric: average daily increment per 100k

JHU counts are cumulative, so the new cases over the last N days is simply (cumulative today − cumulative N days ago). Dividing by N gives a smoothed average daily increment — this is what flattens out weekend dips and reporting spikes. We then normalize per 100,000 inhabitants so countries of any size sit on the same scale.

Show code 
N = 7  # one week — long enough to smooth reporting noise, short enough to stay current

latest, prior = cases.columns[-1], cases.columns[-1 - N]

cases["NdayIncrement"] = cases[latest] - cases[prior]
cases["DailyAvg"]      = cases["NdayIncrement"] / N
cases["DailyAvgPer100k"] = cases["DailyAvg"] / cases["Population"] * 100_000

ranking = cases.sort_values("DailyAvgPer100k", ascending=False)
ranking[["Population", "DailyAvg", "DailyAvgPer100k"]].head(10).round(1)

How did the outbreak evolve?

A single snapshot hides the dynamics. Here is the same per-100k metric plotted as a trend over time for a handful of comparable countries — revealing the timing and height of each wave rather than a single number.

Show code 
picks = ["Czechia", "Sweden", "Germany", "Italy"]

daily_new = cases.loc[picks, cases.columns[1:-3]].diff(axis=1)  # de-cumulate
per100k   = daily_new.div(population.loc[picks], axis=0) * 100_000
trend     = per100k.T.rolling(N).mean().tail(120)

fig, ax = plt.subplots()
for country in picks:
    ax.plot(trend.index, trend[country], label=country, linewidth=2)
ax.set_ylabel("Daily new cases / 100k (7-day avg)")
ax.legend(frameon=False)
fig.autofmt_xdate()
plt.tight_layout()
plt.show()

The trajectories tell the story the ranking can’t: Italy’s tall, sustained wave through late 2022; Germany’s slower decline; Czechia’s brief sharp spike followed by a return to one of the lower per-capita rates in the group.

Takeaways

  • Normalize before you compare. Per-100k rates and absolute counts produce almost opposite country rankings; only the former is meaningful across borders.
  • Smooth, don’t snapshot. A 7-day average increment removes the weekday reporting artefacts that make raw daily figures so noisy.
  • Snapshot and trend. A ranking answers “who’s worst right now”; the time series answers “and is it getting better or worse” — you need both.

## Joining population

The UN file is keyed by country name with population in **thousands**, so we
multiply by 1,000 to get absolute people. We attach it as a new column and drop
micro-states (population < 1,000) whose tiny denominators produce unstable rates.

```{python}
population = (pd.read_csv("populations.csv")

cases.insert(0, "Population", population)
cases = cases[cases["Population"] > 1000]
cases[["Population"]].loc[["Czechia", "Germany", "Sweden"]]
```

## The core metric: average daily increment per 100k

JHU counts are **cumulative**, so the new cases over the last `N` days is simply
*(cumulative today − cumulative N days ago)*. Dividing by `N` gives a smoothed
**average daily increment** — this is what flattens out weekend dips and reporting
spikes. We then normalize per 100,000 inhabitants so countries of any size sit on
the same scale.

```{python}
N = 7  # one week — long enough to smooth reporting noise, short enough to stay current

latest, prior = cases.columns[-1], cases.columns[-1 - N]

cases["NdayIncrement"] = cases[latest] - cases[prior]
cases["DailyAvg"]      = cases["NdayIncrement"] / N
cases["DailyAvgPer100k"] = cases["DailyAvg"] / cases["Population"] * 100_000

ranking = cases.sort_values("DailyAvgPer100k", ascending=False)
ranking[["Population", "DailyAvg", "DailyAvgPer100k"]].head(10).round(1)
```

## How did the outbreak evolve?

A single snapshot hides the dynamics. Here is the same per-100k metric plotted as a
**trend over time** for a handful of comparable countries — revealing the timing and
height of each wave rather than a single number.

```{python}
#| label: fig-trend
#| fig-cap: "Daily new cases per 100k (7-day rolling average), last 120 days."
picks = ["Czechia", "Sweden", "Germany", "Italy"]

daily_new = cases.loc[picks, cases.columns[1:-3]].diff(axis=1)  # de-cumulate
per100k   = daily_new.div(population.loc[picks], axis=0) * 100_000
trend     = per100k.T.rolling(N).mean().tail(120)

fig, ax = plt.subplots()
for country in picks:
    ax.plot(trend.index, trend[country], label=country, linewidth=2)
ax.set_ylabel("Daily new cases / 100k (7-day avg)")
ax.legend(frameon=False)
fig.autofmt_xdate()
plt.tight_layout()
plt.show()
```

The trajectories tell the story the ranking can't: Italy's tall, sustained wave
through late 2022; Germany's slower decline; Czechia's brief sharp spike followed by
a return to one of the lower per-capita rates in the group.

## Takeaways

- **Normalize before you compare.** Per-100k rates and absolute counts produce
  almost opposite country rankings; only the former is meaningful across borders.
- **Smooth, don't snapshot.** A 7-day average increment removes the weekday
  reporting artefacts that make raw daily figures so noisy.
- **Snapshot _and_ trend.** A ranking answers "who's worst right now"; the time
  series answers "and is it getting better or worse" — you need both.
